# RAG 功能设置指南

## 概述

RAG (Retrieval-Augmented Generation) 功能已成功集成到 AI Chat 系统中。现在 AI 助手可以基于系统知识库回答关于系统功能、使用方法等问题。

## 功能特性

### 1. 知识库管理
- 支持添加、删除、搜索知识库文档
- 文档类型：文档、API、指南、FAQ
- 支持标签和分类管理
- 公共文档和私人文档权限控制

### 2. 智能检索
- 基于向量相似度的文档检索
- 支持 Ollama 嵌入模型（推荐：nomic-embed-text）
- 备选文本相似度算法

### 3. 增强对话
- 聊天界面中的 RAG 开关
- 基于检索文档的上下文增强回答
- 显示回答来源和相似度

## 设置步骤

### 1. 数据库迁移
```bash
# 执行 RAG 表创建脚本
psql -d your_database -f src/scripts/migrate-rag-tables.sql
```

### 2. 安装嵌入模型（推荐）
```bash
# 安装 Ollama 嵌入模型
ollama pull nomic-embed-text
```

### 3. 初始化系统知识库
1. 访问 `/rag-admin` 页面
2. 点击"初始化系统知识库"按钮
3. 系统会自动添加基础文档

### 4. 测试 RAG 功能
1. 打开 AI 聊天界面
2. 启用"使用 RAG 增强"开关
3. 询问系统相关问题，如：
   - "如何上传文件？"
   - "怎么创建数据集？"
   - "API 密钥怎么使用？"

## 使用方法

### 管理知识库
访问 `/rag-admin` 页面可以：
- 添加新的知识库文档
- 搜索现有文档
- 删除不需要的文档
- 查看文档详情

### 聊天中使用 RAG
1. 打开 AI 聊天对话框
2. 在输入框上方找到"使用 RAG 增强"开关
3. 启用开关后，AI 会基于知识库回答问题
4. 回答会包含相关文档的引用信息

## 技术实现

### 后端 API
- `src/server/routes/rag.ts` - RAG 路由
- `src/server/services/ragService.ts` - RAG 服务

### 前端组件
- `src/app/rag-admin/page.tsx` - 知识库管理界面

### 数据库表
- `knowledge_base` - 知识库文档
- `rag_queries` - RAG 查询历史

## 故障排除

### 嵌入模型问题
如果 `nomic-embed-text` 模型不可用，系统会自动使用备选的文本哈希算法，但效果会有所降低。

### 权限问题
确保用户有适当的权限访问知识库文档。系统文档默认为公共可见。

### 性能优化
对于大量文档，考虑：
- 添加数据库索引
- 实现文档分块
- 使用专门的向量数据库（如 Pinecone、Weaviate）
