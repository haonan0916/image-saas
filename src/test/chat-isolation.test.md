# 聊天对话隔离测试指南

## 问题描述
之前的聊天系统存在对话隔离问题：
- 不同对话框之间的"AI正在思考"状态会互相干扰
- 多个对话同时进行时，流式响应可能混乱
- 全局状态管理导致状态泄露

## 修复内容

### 1. 状态隔离
- 将 `isLoading` 从单一状态改为 `Map<string, boolean>` 管理每个会话的加载状态
- 每个会话有独立的加载状态，不会互相影响

### 2. 消息ID唯一性
- 临时消息ID包含会话ID：`temp-user-${timestamp}-${sessionId}`
- 确保不同会话的临时消息不会冲突

### 3. 流式响应验证
- 后端流式响应包含 `sessionId` 和 `requestId`
- 前端验证响应的会话ID是否匹配当前会话
- 只更新匹配的会话数据

### 4. 状态清理
- 切换会话时清理其他会话的加载状态
- 对话框关闭时清理所有状态
- 删除会话时清理相关状态

## 测试步骤

### 测试1：多对话框隔离
1. 打开第一个聊天对话框
2. 发送消息，观察"AI正在思考"状态
3. 不要等待回复完成，立即打开第二个聊天对话框
4. 在第二个对话框发送消息
5. 验证：两个对话框的加载状态应该独立，不会互相影响

### 测试2：会话切换
1. 在一个对话框中创建多个会话
2. 在会话A发送消息，立即切换到会话B
3. 在会话B发送消息
4. 验证：每个会话的加载状态独立，消息不会串到其他会话

### 测试3：对话框关闭清理
1. 打开对话框，发送消息
2. 在AI回复过程中关闭对话框
3. 重新打开对话框
4. 验证：之前的加载状态已清理，不会显示"正在思考"

## 预期结果
- ✅ 不同对话框之间完全隔离
- ✅ 每个会话有独立的加载状态
- ✅ 流式响应不会串到错误的会话
- ✅ 状态清理机制正常工作
